C'est le cœur du réacteur de ton application. Pour Panthers CrossFit, on utilise une architecture hybride : Firestore est le "cerveau" dans le cloud, et Hive est la "mémoire immédiate" dans le téléphone.Voici le rappel du fonctionnement étape par étape :1. La Première Connexion (Le "Pont")Lorsqu'un nouveau membre ou un staff se connecte pour la toute première fois :Action : Il saisit son matricule.Firestore : L'app va chercher le document correspondant dans la collection members.Hive : Dès que les infos arrivent (Nom, Rôle, Date d'expiration), l'app les transforme en objet MemberAccount et les enregistre dans une box chiffrée (AES-256).Résultat : Le téléphone possède désormais une copie locale sécurisée de l'identité de l'utilisateur.2. Le Lancement Quotidien (Le "Zéro Friction")Une fois que la première connexion est faite, voici ce qui se passe quand on ouvre l'app le lendemain :Étape 1 : L'app regarde dans Hive.Étape 2 : Elle trouve le matricule et le rôle. Elle n'attend pas de réponse d'internet.Étape 3 : Elle affiche instantanément le Dashboard (ou le QR Code).Avantage : Même si le membre est dans une zone où le réseau capte mal devant la salle, son QR Code s'affiche sans charger.3. La Vérification d'Éligibilité (Le "Gardien")Pour savoir si le QR Code doit être affiché ou caché :L'application compare l'heure locale (DateTime.now()) avec la dateExpiration stockée dans Hive.Si OK : Le QR Code apparaît.Si Expiré : Le QR Code est remplacé par le message de rappel et le bouton de paiement.4. Les Modifications et le mode Offline (Le Staff)C'est ici que la magie de Firestore opère pour ton staff :Action : Le staff modifie une info client (ex: changement de téléphone).Si Internet est présent : La modif est envoyée à Firestore ET mise à jour dans le Hive local.Si Internet est absent : Firestore garde la modification "en attente" dans son propre cache. Dès que la tablette retrouve du Wi-Fi, elle envoie tout au cloud automatiquement. Le staff n'a rien à faire.5. Le Super Admin, c'est le "Grand Patron" de l'application. Contrairement au Staff qui est sur le terrain pour gérer les entrées et les inscriptions, le Super Admin a une vision stratégique et financière du club.Voici ses pouvoirs exclusifs dans ton système hybride (Firestore + Hive) :1. Contrôle Total des RôlesAction : Lui seul peut transformer un "Membre" en "Staff" ou nommer un autre "Super Admin".Sécurité : Dans l'interface, le menu déroulant des rôles n'apparaît que pour lui. Dans Firestore, les Security Rules rejettent toute tentative de modification de rôle venant d'un compte "Staff".2. Gestion Financière (Le Chiffre d'Affaires)Vue exclusive : Il possède un tableau de bord (Dashboard) que le Staff ne peut pas voir.Données : Il voit la somme totale des abonnements perçus, les statistiques de vente (Airtel vs Orange Money) et les prévisions pour le mois suivant.Modification des prix : Si tu décides de changer le tarif du forfait "CrossFit Premium", seul le Super Admin peut modifier cette valeur dans la collection settings de Firestore.3. Droit de "Vie ou de Mort" sur les ComptesSuppression : Le Staff peut inscrire un membre, mais il ne peut pas le supprimer. Seul le Super Admin peut cliquer sur le bouton "Supprimer".Bannissement : Il peut forcer la déconnexion d'un membre à distance en changeant son statut dans le Cloud. Lors de la prochaine synchronisation, le Hive du membre se videra et le QR Code disparaîtra.4. Supervision du StaffJournal d'activité : Il peut voir quel membre du staff a inscrit quel client.
